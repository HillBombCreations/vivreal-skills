## Architecture — 12 Lambdas (main SAM stack)

`userAndAuth` (profile switch, email, logout — email update now fails closed on Cognito ListUsers truncation and rejects an email held by another user), `billingAndSubscription` (Stripe checkout/portal/webhooks/preflight/overage), `createAndJoinGroup`, `createSites` (templates + template instantiation, Route53, domain purchase/transfer, Step Functions, WebSockets — most files + AWS integrations; 2048MB/60s since the instantiation work), `getGroupInformation` (read-only + admin attribution + `GET /api/analytics/site/traffic`; serializes `overQuotaSinceCancellation` + derived `usage.{cdn,api,agent}.{pctOfQuota,nearQuota,overQuota}`), `updateGroup` (members, roles, API keys, integrations, overage toggle, featureFlags, `refreshSocialProfile`), `agent` (`src/agent/` — AI agent orchestrator + tool layer), `webhookDelivery` (SQS consumer, HMAC-signed outbound webhooks), `analyticsSnapshot` (EventBridge `cron(0 6 * * ? *)`), `squareTokenRefresh` (daily sweep cron), `squareRefreshOne` (invoke-only — cross-stack target for VR_Client_API's `squareTokenGuard`), `instantiateTemplateWorker` (NEW — **no HTTP route**; direct-invoked by createSites' `instantiateTemplate` via the `INSTANTIATE_TEMPLATE_WORKER_LAMBDA` env; 1024MB/900s; own DLQ + `InstantiateTemplateWorkerErrorsAlarm`). **The websocket stack (`websocket/template.yaml`) is separate — never count its 4 functions in this roster**; `allRoutes.packaged.yaml` is a stale CI artifact — never count from it. 4 of 12 have WebSocket integration (agent, createAndJoinGroup, createSites, updateGroup). Shared: `encryptionUtils.js` (AES-256-GCM for integration creds), `handleHBRoutes.js`, `socket.js`, `auditLog.js`, `deriveDbKey.js`.

## Square P2 (token lifecycle + provider mutex)

- `squareTokenRefresh`: daily `rate(24 hours)` sweep refreshing tokens <14 days from Square's 30-day hard expiry; 36h dead-man + Errors≥1 alarms.
- `squareRefreshOne`: no route/schedule; resource-based `AWS::Lambda::Permission` for VR_Client_API's role (`SquareRefreshOneInvokerRoleArn`) — synchronous single-account refresh.
- **D4 payments-provider mutex**: one active payments provider per group — `updateIntegrations.js` + `oauthService.js:storeTokens` throw `PaymentsProviderAlreadyActive`.
- Revoke-on-disconnect via `src/shared/square/revokeSquareToken.js`. Feature gate: `group.featureFlags.squareStorefront` (write path `PUT /api/group/featureFlags` — see gotchas).

## Tier quotas — v3.0.0 sentinel scheme + AI agent

Quotas live in **`@hillbombcreations/tier-quotas` v3.0.0** — **sentinel scheme: -1 = unlimited, 0 = no access (fail-closed), >0 = cap**, read via helpers `isUnlimited(<0)`, `hasAccess(!==0)`, `getEffectiveLimit`; the old `isUnlimitedQuota(0)==unlimited` inversion is GONE. `agentActions`: free 0 (**now truly denies** — was a dead gate, same fix for free `webhooks: 0`), basic 50, pro 500, **proplus 500 (cut from 5,000 in v2.3.0, retained in v3.0.0)**, enterprise -1. **W6 read-flip:** the 6 tier-driven fields (entries/seats/sites/apiCalls/cdnBytes/agentActions) read live from `getTierQuotas(group.tier)` at every gate (`joinGroup`, `acceptRequest`, agent metering, `getAgentContext`, `instantiateTemplate`, `createSiteCollectionData`, `createSiteWithExistingCollections`, `getGroupInfo` v1+legacy, `updateEmail` serializers) — `updateGroupTier` + `handleSubscriptionEnd` STOPPED writing them, and the per-group `agentUsage.quota` override is GONE from `checkAgentQuota` (`src/agent/services/metering.js`): past-quota agent use requires `overageBilling.enabled` and hard-stops at the spending cap. `mediaUsage.quota` is admin-override-only (W7). The `aiActionsEnabled` toggle requires `hasAccess(agentActions)` → new `FeatureNotAvailableOnTier` error. Allowlist gate + prompt caching + monthly usage reset live in `src/agent/`.

## Multi-tenant routing — `dbKey` (NOT `key`)

**Secure reads `req.query.dbKey`** — CMS reads `key`. The portal proxy must set `dbKey` explicitly for Secure routes. `dynamicDb.js` pools per-tenant connections (`maxPoolSize=3`). mainDb `Vivreal` holds group metadata. **Never filter mainDb by `groupName`** — use `{ _id: groupID }` / `{ key: dbKey }`.

## Adding a route — 7 steps (step 7 = the 502 trap)

1. `services/foo.js` → 2. `services/index.js` → 3. `api/controllers/foo.js` (set `req.resData`) → 4. `api/controllers/index.js` → 5. `scripts/validators.js` (Joi) → 6. `api/index.js` (`handleHBRoutes`) → **7. `cloudformation/<lambda>.yaml` API Gateway event**.

**Step 7 is a hard rule.** Secure (like CMS + Client) registers **one explicit API Gateway event per route** — there is **no catch-all `{proxy+}` integration**. An Express route the gateway has no event for is unreachable: the request falls through and fails — **502** post-deploy here (CMS surfaces the same miss as a 403; Client rejects it at the edge). Always add the event in the `cloudformation/` fragment; `cloudYamls/allRoutes.yaml` is generated by `scripts/merge-template.js` — never edit it directly. (Cross-repo rule + why Main API is exempt: `vivreal-lambda` deploy reference + `vivreal-auth-architecture`.)

## Billing specifics

- **Downgrade preflight** (`GET /api/billing/preflightDowngrade`) — recounts `collection_objects` + `webhooks` via tenant connection, compares to target tier quotas, returns `{ ok, violations[], currentTier, targetTier }`. Downgrade to a 0-access target (e.g. free's `webhooks`/`agentActions` 0) is now a preflight violation. Re-used by `createCheckoutSession` as a defence-in-depth gate (400 with violations on reject). Opportunistically heals `entries.totalSize` drift.
- **Cancellation** (`handleSubscriptionEnd.js`) — demotes to `free` (it no longer writes the 6 tier-driven quota fields — they derive live from the tier), sets `frozen:true` + persists `overQuotaSinceCancellation {violations, setAt}` when over-limit; `getGroupInfo` serializes it → drives the portal frozen/over-quota banner. Webhook-order safe: lookup by `stripeSubscriptionID` misses (no-op) when cancellation is the side-effect of an in-progress tier change.
- **Stripe portal lockdown** — optional `STRIPE_PORTAL_CONFIGURATION_ID` disables `subscription_update` so tier changes route through the preflight-gated Manage Plan page. Bootstrap is a manual one-time script.
- **Overage billing** — new paid subs **auto-enroll** `overageBilling.enabled` + the tier default spending cap (`DEFAULT_SPENDING_CAP`; grandfathered if the prior tier was overage-eligible, enterprise excluded); toggle via NEW **`PUT /api/group/overageBilling`** (`toggleOverageBilling.js`, native-driver persist). `createCheckoutSession` discloses overage + cap in Stripe `custom_text`. `handleInvoiceCreated.js` adds Stripe invoice items at the tier-quotas `OVERAGE_RATES` (CDN $0.50/GB, API $0.005/call, agent $0.05/action); the `overageBilling.lastInvoice` snapshot persists via the native driver, and a **deterministic per-(invoice,bucket) Stripe idempotency key** stops duplicate charges on `invoice.created` retries; fails safe (undercharge not overcharge). **GOTCHA: `overageBilling` + `stripeSubscriptionID` are UNDECLARED in strict `groupSchema` (v1.24.0/1.25.0) — writes go through the raw MongoDB driver pending a schemas PR.**

- The orphaned-OTEL-layer gotcha is RESOLVED — `cloudformation/updateGroup.yaml`'s `Layers` now carries only the ffmpeg ARN via `/vivreal/prod/shared/ffmpeg-arn` SSM; no `AWS_LAMBDA_EXEC_WRAPPER`.
- Former P0 security gaps are CLOSED + test-locked (SEC-1..3): Stripe webhook signature IS verified (`stripe.webhooks.constructEvent` in `handleEvents.js`, raw body captured via `express.json({ verify })` in `createApp.js`); `acceptRequest`/`denyRequest` inherit the Cognito default authorizer + owner/admin 403; RBAC enforced on write ops (`UnauthorizedRoleException`); CORS is single-layer (`createApp.js`). Legit `Authorizer: NONE` remains only on the Stripe webhook + OAuth callback.
- Deps: `@hillbombcreations/schemas ^1.24.0`, `tier-quotas ^3.0.0`, `site-loader ^0.1.3` (was a `file:` dep — now semver from GitHub Packages), `site-renderer` pinned EXACTLY `1.31.0` (byte-match the fleet renderer — do not loosen without regenerating `renderer.capability.json`).
